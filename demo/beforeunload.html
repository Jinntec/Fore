showing page exit confirmation

This feature allows to show/not show a confirmation based on data changed by the user.

Confirmation based on a condition

On show-confirmation attribute you can specify an XPath resolving to a boolean value that determines whether to show/not show a page exit warning when the user navigates away from the page e.g. by clicking a link.

Clicking the link without touching the control will just send you to the destination.

If you change the control value a page exit warning will appear allowing the user to cancel navigation, go back and save changes.

Show confirmation when data changed

With just the attribute given the confirmation will show once the data have been changed by user interaction.

show-confirmation can be empty, the empty string, 'true' or 'show-confirmation'

fx-setvalue or other actions changing data will have no direct effect on wether to show/not show the confirmation. It's just answers the question if the user has at least changed one control value.
