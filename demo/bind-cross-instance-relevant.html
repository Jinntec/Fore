Fixed: fx-bind[relevant] across instances now reactively updates

A bind's relevant (and, via the same code path, calculate/ required/constraint) facet reactively recomputes on a plain fx-control value change — including when the expression references a node in a different fx-instance than the bind's own ref. Previously that cross-instance case was evaluated once at load and then never again, no matter how many times the referenced value changed.

Root cause: fx-bind.js's _buildBindGraph() resolved instanceId once from the bind's own ref, then reused that same id to canonicalize the path of every node referenced inside relevant (via _addDependencies). For a cross-instance reference this produced a canonical path that didn't match the real node's path anywhere else in the model (e.g. getPath(queryNode, 'items') instead of getPath(queryNode, 'ui')), so model.mainGraph.dependantsOf(realChangedPath) never found the bind as a dependant, and compute() was never called for it again.

Fix: _addDependencies now resolves each referenced node's own instance id via model.getInstanceIdForNode() — a reverse lookup from the node's ownerDocument to the owning fx-instance's id — instead of reusing the bind's own instance id for every reference.
