Data - a home for your precious data

First of all - there can as many data elements as you need.

Description

We got 2 data elements that create 2 separate XML documents. The first in document order is the default data element. All following ones need to have an id.

It's important to understand that the data are not part of the html page but exist as separate document in memory. Therefore you cannot inspect the state of the data with a developer tool.

This not only ensures the integrity of the data but also allows us to work with real XML documents.

Limitations of inline data

Inline data use the HTML DOM of the browser which e.g. only supports lowercase element names, no namespaces etc. If you intend to work with full XML documents you need to use external to use external files with the src attribute. See next example.

Addressing data elements

To refer to a node in a non-default data element you have to use the $default function which takes the id as parameter. This returns the root-node of the data.
