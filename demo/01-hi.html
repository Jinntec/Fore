Lazy Mode

In lazy mode you don't even need to create a model. This is for the 'small and quick' use cases where you don't need much logic or constraints.

Description

Fore is implemented as Web Components. A form in Fore is always wrapped up in a fx-fore element. Besides that we have a fx-output which - well - outputs a referenced data node.

Fore is MVC but where's the model? It gets auto-generated if you don't provide one. For each Fore component with a ref a node in the instance data is created. The value of that bound component is taken as the default value. The form will create a XML document for your data.
