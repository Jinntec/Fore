Lazy Greeting

Description

Fore is implemented as Web Components. A form in Fore is always wrapped up in a fx-form element. Besides that we have a fx-output which - well -outputs a referenced data node.

Fore is MVC but where's the model? It gets auto-generated if you don't provide one. For each Fore component with a ref a node in the instance data is created. The value of that bound component is taken as the default value. The form will create a XML document for your data.

If you haven't felt that lazy this morning you could have written the equivalent like so:

Hello Universe

The form holds a single fx-model which itself contains a fx-instance element. This is where our actual data are living. A form can contain a single model but as many fx-instance elements as you like.
